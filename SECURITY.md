# Security Policy

## Supported versions

This project is currently an experimental MVP. Security and privacy-related reports are welcome, but the project should not be treated as a production-grade security or compliance product.

## Reporting a vulnerability or privacy issue

Please do **not** open public issues containing real personal data, confidential documents, token dictionaries, screenshots with personal data, or production review reports.

Use synthetic examples only.

If you need to describe a privacy issue, replace all real values with fictional but structurally similar values.

## Sensitive data

Do not share:

- real input documents;
- real output documents;
- `project_dictionary.json`;
- `dictionary.json`;
- `rules/manual_hide.txt`;
- `rules/manual_allow.txt`;
- `feedback/cases.jsonl` generated from real documents;
- `review/review_cases.xlsx` generated from real documents;
- reports containing original values.

## Scope

BeforeSending is a local pseudonymization / masking / risk-reduction utility.

It does not guarantee complete anonymization or regulatory compliance.

## Privacy incident prevention

Issues, pull requests or discussions containing real personal or confidential data may be removed or edited to protect privacy.
